# Бизнес-правила и Логика

## 1. Подписка и Доступ

- Доступ к каналам предоставляется **только** пользователям с активной подпиской (запись в таблице `user_subscriptions` с `datetime_end_subscribe` в будущем).
- Срок подписки считается до конкретной даты и времени (`datetime_end_subscribe`).
- При истечении подписки пользователь автоматически банится из всех каналов и разбанивается для возможности вернуться.

## 2. Продление и Оплата

- Оплата подтверждается **вручную** администратором по скриншоту.
- **Логика продления подписки:** При подтверждении оплаты подписка продлевается на указанное количество дней от текущей даты окончания (или создается новая, если подписки не было).
- Тарифы:
  - 1 месяц: 1500 рублей
  - 3 месяца: 4275 рублей (-5%)
  - 6 месяцев: 8100 рублей (-10%)
  - 12 месяцев: 15300 рублей (-15%)
- Промокоды могут предоставлять дополнительную скидку.

## 3. Уведомления и Исключение (Ban/Kick)

- Бот проверяет подписки через планировщик (APScheduler).
- **3 дня до конца:** Отправляется напоминание пользователю.
- **1 день до конца:** Отправляется напоминание пользователю.
- **Срок истек:**
    1. Подписка удаляется из БД (`user_subscriptions`)
    2. Пригласительные ссылки удаляются (`user_links`)
    3. Пользователь банится из всех каналов (`forward_channel_id`)
    4. Пользователь сразу разбанивается (unban), чтобы иметь возможность вернуться после оплаты

## 4. Каналы

- **channel_id** - ID канала-источника (откуда копировать через юзербот)
- **forward_channel_id** - ID канала-назначения (куда копировать, куда давать доступ пользователям)
- При добавлении канала:
  - Создается запись в `tracked_channels`
  - Создается новый столбец в `user_links` для хранения ссылок этого канала
  - Для всех активных пользователей генерируются пригласительные ссылки
- При удалении канала:
  - Удаляется запись из `tracked_channels`
  - Удаляется столбец из `user_links`

## 5. Пригласительные ссылки

- Для каждого канала создается одноразовая пригласительная ссылка (`member_limit=1`)
- Ссылки генерируются через `bot.create_chat_invite_link()`
- Ссылки хранятся в таблице `user_links` (динамические столбцы по channel_id)
- При истечении подписки ссылки удаляются

## 6. Планировщик задач

- Используется APScheduler для автоматических задач
- Задачи хранятся в БД (`tasks_for_scheduler`) для восстановления при перезапуске
- Типы задач:
  - `end_subscribe` - бан пользователя при истечении подписки
  - `reminder_1day` - напоминание за 1 день до окончания
  - `reminder_3days` - напоминание за 3 дня до окончания

## 7. Админ-панель

- Доступ только для пользователей из списка `ADMINS` в конфиге
- Функции:
  - Добавление/удаление каналов
  - Продление подписки пользователям
  - Подтверждение оплат
  - Создание промокодов
  - Просмотр списка пользователей и платежей
  - Экспорт пользователей в CSV
  - Массовые уведомления

## 8. Защита (Failsafe)

- При добавлении канала проверяется, что бот является участником канала-назначения
- При создании ссылок обрабатываются ошибки (RetryAfter, блокировки)
- При бане пользователя обрабатываются ошибки (пользователь уже забанен, бот не админ и т.д.)
