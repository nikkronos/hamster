# Архитектура и Структура Проекта

Проект разделен на модули для удобства поддержки и расширения.

## Структура файлов

*   **`bot.py`**: Точка входа. Инициализирует базу данных, запускает планировщик задач и начинает опрос серверов Telegram (polling).
*   **`data/config.py`**: Отвечает за загрузку переменных окружения (токен, ID админов, API_ID, API_HASH) из файла `.env`.
*   **`loader.py`**: Создает основные объекты: `bot` (экземпляр Bot), `dp` (Dispatcher), `scheduler` (AsyncIOScheduler). Используется для избежания циклических импортов.
*   **`database/`**: Содержит всю логику работы с SQLite:
    *   **`keys.py`**: Структура таблиц БД
    *   **`work_db.py`**: Подключение к БД и выполнение запросов
    *   **`getters.py`**: Функции получения данных из БД
    *   **`setters.py`**: Функции изменения данных в БД
*   **`utils/`**: Утилиты:
    *   **`misc.py`**: Бизнес-логика (подписки, ссылки, бан/разбан, планировщик)
    *   **`states_users.py`**: Работа с FSM состояниями пользователей
    *   **`record_logs.py`**: Логирование
*   **`handlers/`**: Пакет с обработчиками сообщений, разделенными по функциональности:
    *   **`start.py`**: Обработчик команды /start
    *   **`about_subscribe.py`**: Информация о подписке и тарифах
    *   **`pay.py`**: Процесс оплаты (скриншот, отправка админу)
    *   **`check_subscribe.py`**: Проверка статуса подписки
    *   **`channel_list.py`**: Список каналов с пригласительными ссылками
    *   **`promocode.py`**: Работа с промокодами
    *   **`admins/`**: Админские функции:
        *   `menu.py` - админ-меню
        *   `add_channel.py` - добавление канала
        *   `delete_channel.py` - удаление канала
        *   `add_subscription.py` - продление подписки
        *   `confirm_pay.py` - подтверждение оплаты
        *   `create_promocode.py` - создание промокода
        *   `user_list.py` - список пользователей
        *   `payment_list.py` - список платежей
    *   **`author/`**: Функции автора (управление БД, логи, обновления)
*   **`keyboards/`**: Клавиатуры для интерфейса:
    *   `keyboards.py` - базовые функции создания клавиатур
    *   `users.py` - клавиатуры для пользователей
    *   `admin.py` - клавиатуры для админов
    *   `developer.py` - клавиатуры для разработчика
*   **`middlewares/`**: Middleware для обработки сообщений:
    *   `filter_banned_users.py` - фильтрация забаненных пользователей

## База Данных

Используется **SQLite**. Файл `database/database.sqlite` создается в папке `database/`.

### Таблицы:

1.  **`users`**: Информация о пользователях
    - `user_id` (INTEGER PRIMARY KEY)
    - `first_name` (TEXT)
    - `username` (TEXT)
    - `date_reg` (TIMESTAMP)

2.  **`tracked_channels`**: Отслеживаемые каналы
    - `id` (INTEGER PRIMARY KEY)
    - `channel_id` (INTEGER) - ID канала-источника
    - `title` (TEXT) - название канала
    - `forward_channel_id` (INTEGER) - ID канала-назначения

3.  **`user_subscriptions`**: Подписки пользователей
    - `id` (INTEGER PRIMARY KEY)
    - `user_id` (INTEGER)
    - `datetime_end_subscribe` (TIMESTAMP)

4.  **`user_links`**: Пригласительные ссылки пользователей
    - `user_id` (INTEGER PRIMARY KEY)
    - Динамические столбцы для каждого канала (название = channel_id)

5.  **`payments`**: Платежи
    - `id` (INTEGER PRIMARY KEY)
    - `user_id` (INTEGER)
    - `amount` (TEXT)
    - `promocode` (TEXT)
    - `is_paid` (INTEGER)
    - `date` (TIMESTAMP)

6.  **`promocodes`**: Промокоды
    - `id` (INTEGER PRIMARY KEY)
    - `promocode` (TEXT)
    - `percent` (INTEGER)
    - `expiration` (TIMESTAMP)

7.  **`tasks_for_scheduler`**: Задачи для планировщика
    - `id` (INTEGER PRIMARY KEY)
    - `user_id` (INTEGER)
    - `run_date` (TIMESTAMP)
    - `name` (TEXT) - тип задачи (end_subscribe, reminder_1day, reminder_3days)

## Планировщик задач

Используется **APScheduler** (AsyncIOScheduler) для:
- Автоматического бана пользователей при истечении подписки
- Отправки напоминаний об окончании подписки (за 3 дня и 1 день)
- Восстановления задач из БД при перезапуске бота

## Работа с каналами

### Структура:
- **channel_id** - ID канала-источника (откуда копировать через юзербот)
- **forward_channel_id** - ID канала-назначения (куда копировать, куда давать доступ пользователям)

### Процесс:
1. Админ добавляет канал через админ-панель
2. Для каждого активного пользователя создается пригласительная ссылка
3. Пользователь получает доступ к каналу по ссылке
4. При истечении подписки пользователь банится из всех каналов

## Юзербот

**Примечание:** Код юзербота (Pyrogram) не находится в этом репозитории. Юзербот работает отдельно и копирует содержимое каналов.

## Потоки данных

### Процесс оплаты:
1. Пользователь выбирает тариф
2. Получает инструкцию по оплате
3. Отправляет скриншот оплаты
4. Админ получает уведомление с кнопками подтверждения
5. При подтверждении создается/продлевается подписка
6. Генерируются пригласительные ссылки
7. Пользователь получает доступ к каналам

### Процесс истечения подписки:
1. Планировщик проверяет подписки
2. За 3 дня до окончания - отправка напоминания
3. За 1 день до окончания - отправка напоминания
4. При истечении - бан из всех каналов, удаление подписки, разбан для возможности вернуться
